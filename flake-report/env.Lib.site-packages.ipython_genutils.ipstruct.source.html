258 def merge(self, __loc_data__=None, __conflict_solve=None, **kw):
259 """Merge two Structs with customizable conflict resolution.
260  
261 This is similar to :meth:`update`, but much more flexible. First, a
262 dict is made from data+key=value pairs. When merging this dict with
263 the Struct S, the optional dictionary 'conflict' is used to decide
264 what to do.
265  
266 If conflict is not given, the default behavior is to preserve any keys
267 with their current value (the opposite of the :meth:`update` method's
268 behavior).
269  
270 Parameters
271 ----------
272 __loc_data : dict, Struct
273 The data to merge into self
274 __conflict_solve : dict
275 The conflict policy dict. The keys are binary functions used to
276 resolve the conflict and the values are lists of strings naming
277 the keys the conflict resolution function applies to. Instead of
278 a list of strings a space separated string can be used, like
279 'a b c'.
280 kw : dict
281 Additional key, value pairs to merge in
282  
283 Notes
284 -----
285  
286 The `__conflict_solve` dict is a dictionary of binary functions which will be used to
287 solve key conflicts. Here is an example::
288  
289 __conflict_solve = dict(
290 func1=['a','b','c'],
291 func2=['d','e']
292 )
293  
294 In this case, the function :func:`func1` will be used to resolve
295 keys 'a', 'b' and 'c' and the function :func:`func2` will be used for
296 keys 'd' and 'e'. This could also be written as::
297  
298 __conflict_solve = dict(func1='a b c',func2='d e')
299  
300 These functions will be called for each key they apply to with the
301 form::
302  
303 func1(self['a'], other['a'])
304  
305 The return value is used as the final merged value.
306  
307 As a convenience, merge() provides five (the most commonly needed)
308 pre-defined policies: preserve, update, add, add_flip and add_s. The
309 easiest explanation is their implementation::
310  
311 preserve = lambda old,new: old
312 update = lambda old,new: new
313 add = lambda old,new: old + new
314 add_flip = lambda old,new: new + old # note change of order!
315 add_s = lambda old,new: old + ' ' + new # only for str!
316  
317 You can use those four words (as strings) as keys instead
318 of defining them as functions, and the merge method will substitute
319 the appropriate functions for you.
320  
321 For more complicated conflict resolution policies, you still need to
322 construct your own functions.
323  
324 Examples
325 --------
326  
327 This show the default policy:
328  
329 >>> s = Struct(a=10,b=30)
330 >>> s2 = Struct(a=20,c=40)
331 >>> s.merge(s2)
332 >>> sorted(s.items())
333 [('a', 10), ('b', 30), ('c', 40)]
334  
335 Now, show how to specify a conflict dict:
336  
337 >>> s = Struct(a=10,b=30)
338 >>> s2 = Struct(a=20,b=40)
339 >>> conflict = {'update':'a','add':'b'}
340 >>> s.merge(s2,conflict)
341 >>> sorted(s.items())
342 [('a', 20), ('b', 70)]
343 """
