12 Wrapper around stdin which reads (nonblocking) the next available 1024
13 bytes and decodes it.
14  
15 Note that you can't be sure that the input file is closed if the ``read``
16 function returns an empty string. When ``errors=ignore`` is passed,
17 ``read`` can return an empty string if all malformed input was replaced by
18 an empty string. (We can't block here and wait for more input.) So, because
19 of that, check the ``closed`` attribute, to be sure that the file has been
20 closed.
21  
22 :param stdin_fd: File descriptor from which we read.
23 :param errors: Can be 'ignore', 'strict' or 'replace'.
24 On Python3, this can be 'surrogateescape', which is the default.
