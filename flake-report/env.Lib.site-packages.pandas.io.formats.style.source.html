297 the internal default values described in notes.

304 -------

306  
307 Notes

309 Tooltips are created by adding `<span class="pd-t"></span>` to each data cell
310 and then manipulating the table level CSS to attach pseudo hover and pseudo
311 after selectors to produce the required the results.
312  
313 The default properties for the tooltip CSS class are:
314

316 - position: absolute
317 - z-index: 1
318 - background-color: black
319 - color: white
320 - transform: translate(-20px, -20px)

525 **Cell Styles**
526  
527 LaTeX styling can only be rendered if the accompanying styling functions have
528 been constructed with appropriate LaTeX commands. All styling
529 functionality is built around the concept of a CSS ``(<attribute>, <value>)``
530 pair (see `Table Visualization <../../user_guide/style.ipynb>`_), and this
531 should be replaced by a LaTeX
532 ``(<command>, <options>)`` approach. Each cell will be styled individually
533 using nested LaTeX commands with their accompanied options.
534  
535 For example the following code will highlight and bold a cell in HTML-CSS:
536  
537 >>> df = pd.DataFrame([[1,2], [3,4]])
538 >>> s = df.style.highlight_max(axis=None,
539 ... props='background-color:red; font-weight:bold;')
540 >>> s.render()
541  
542 The equivalent using LaTeX only commands is the following:
543  
544 >>> s = df.style.highlight_max(axis=None,
545 ... props='cellcolor:{red}; bfseries: ;')
546 >>> s.to_latex()
547  
548 Internally these structured LaTeX ``(<command>, <options>)`` pairs
549 are translated to the
550 ``display_value`` with the default structure:
551 ``\<command><options> <display_value>``.
552 Where there are multiple commands the latter is nested recursively, so that
553 the above example highlighed cell is rendered as
554 ``\cellcolor{red} \bfseries 4``.
555  
556 Occasionally this format does not suit the applied command, or
557 combination of LaTeX packages that is in use, so additional flags can be
558 added to the ``<options>``, within the tuple, to result in different
559 positions of required braces (the **default** being the same as ``--nowrap``):
560  
561 =================================== ============================================
562 Tuple Format Output Structure
563 =================================== ============================================
564 (<command>,<options>) \\<command><options> <display_value>
565 (<command>,<options> ``--nowrap``) \\<command><options> <display_value>
566 (<command>,<options> ``--rwrap``) \\<command><options>{<display_value>}
567 (<command>,<options> ``--wrap``) {\\<command><options> <display_value>}
568 (<command>,<options> ``--lwrap``) {\\<command><options>} <display_value>
569 (<command>,<options> ``--dwrap``) {\\<command><options>}{<display_value>}
570 =================================== ============================================
571  
572 For example the `textbf` command for font-weight
573 should always be used with `--rwrap` so ``('textbf', '--rwrap')`` will render a
574 working cell, wrapped with braces, as ``\textbf{<display_value>}``.
575  
576 A more comprehensive example is as follows:
577  
578 >>> df = pd.DataFrame([[1, 2.2, "dogs"], [3, 4.4, "cats"], [2, 6.6, "cows"]],
579 ... index=["ix1", "ix2", "ix3"],
580 ... columns=["Integers", "Floats", "Strings"])
581 >>> s = df.style.highlight_max(
582 ... props='cellcolor:[HTML]{FFFF00}; color:{red};'
583 ... 'textit:--rwrap; textbf:--rwrap;'
584 ... )
585 >>> s.to_latex()

589 **Table Styles**
590  
591 Internally Styler uses its ``table_styles`` object to parse the
592 ``column_format``, ``position``, ``position_float``, and ``label``
593 input arguments. These arguments are added to table styles in the format:
594  
595 .. code-block:: python
596  
597 set_table_styles([
598 {"selector": "column_format", "props": f":{column_format};"},
599 {"selector": "position", "props": f":{position};"},
600 {"selector": "position_float", "props": f":{position_float};"},
601 {"selector": "label", "props": f":{{{label.replace(':','§')}}};"}
602 ], overwrite=False)
603  
604 Exception is made for the ``hrules`` argument which, in fact, controls all three
605 commands: ``toprule``, ``bottomrule`` and ``midrule`` simultaneously. Instead of
606 setting ``hrules`` to ``True``, it is also possible to set each
607 individual rule definition, by manually setting the ``table_styles``,
608 for example below we set a regular ``toprule``, set an ``hline`` for
609 ``bottomrule`` and exclude the ``midrule``:
610  
611 .. code-block:: python
612  
613 set_table_styles([
614 {'selector': 'toprule', 'props': ':toprule;'},
615 {'selector': 'bottomrule', 'props': ':hline;'},
616 ], overwrite=False)
617  
618 If other ``commands`` are added to table styles they will be detected, and
619 positioned immediately above the '\\begin{tabular}' command. For example to
620 add odd and even row coloring, from the {colortbl} package, in format
621 ``\rowcolors{1}{pink}{red}``, use:
622  
623 .. code-block:: python
624  
625 set_table_styles([
626 {'selector': 'rowcolors', 'props': ':{1}{pink}{red};'}
627 ], overwrite=False)
628  
629 A more comprehensive example using these arguments is as follows:
630  
631 >>> df.columns = pd.MultiIndex.from_tuples([
632 ... ("Numeric", "Integers"),
633 ... ("Numeric", "Floats"),
634 ... ("Non-Numeric", "Strings")
635 ... ])

637 ... ("L0", "ix1"), ("L0", "ix2"), ("L1", "ix3")
638 ... ])
639 >>> s = df.style.highlight_max(
640 ... props='cellcolor:[HTML]{FFFF00}; color:{red}; itshape:; bfseries:;'
641 ... )
642 >>> s.to_latex(
643 ... column_format="rrrrr", position="h", position_float="centering",
