2 Similar to `PyOS_InputHook` of the Python API, we can plug in an input hook in
3 the asyncio event loop.
4  
5 The way this works is by using a custom 'selector' that runs the other event
6 loop until the real selector is ready.
7  
8 It's the responsibility of this event hook to return when there is input ready.
9 There are two ways to detect when input is ready:
10  
11 The inputhook itself is a callable that receives an `InputHookContext`. This
12 callable should run the other event loop, and return when the main loop has
13 stuff to do. There are two ways to detect when to return:
14  
15 - Call the `input_is_ready` method periodically. Quit when this returns `True`.
