35 class Headers(MutableMapping[str, str]):
36 """
37 Efficient data structure for manipulating HTTP headers.
38  
39 A :class:`list` of ``(name, values)`` is inefficient for lookups.
40  
41 A :class:`dict` doesn't suffice because header names are case-insensitive
42 and multiple occurrences of headers with the same name are possible.
43  
44 :class:`Headers` stores HTTP headers in a hybrid data structure to provide
45 efficient insertions and lookups while preserving the original data.
46  
47 In order to account for multiple values with minimal hassle,
48 :class:`Headers` follows this logic:
49  
50 - When getting a header with ``headers[name]``:
51 - if there's no value, :exc:`KeyError` is raised;
52 - if there's exactly one value, it's returned;
53 - if there's more than one value, :exc:`MultipleValuesError` is raised.
54  
55 - When setting a header with ``headers[name] = value``, the value is
56 appended to the list of values for that header.
57  
58 - When deleting a header with ``del headers[name]``, all values for that
59 header are removed (this is slow).
60  
61 Other methods for manipulating headers are consistent with this logic.
62  
63 As long as no header occurs multiple times, :class:`Headers` behaves like
64 :class:`dict`, except keys are lower-cased to provide case-insensitivity.
65  
66 Two methods support manipulating multiple values explicitly:
67  
68 - :meth:`get_all` returns a list of all values for a header;
69 - :meth:`raw_items` returns an iterator of ``(name, values)`` pairs.
70  
71 """
