107 ``realm`` indicates the scope of protection. It should contain only ASCII
108 characters because the encoding of non-ASCII characters is undefined.
109 Refer to section 2.2 of :rfc:`7235` for details.
110  
111 ``credentials`` defines hard coded authorized credentials. It can be a
112 ``(username, password)`` pair or a list of such pairs.
113  
114 ``check_credentials`` defines a coroutine that checks whether credentials

117  
118 One of ``credentials`` or ``check_credentials`` must be provided but not

123 with the ``create_protocol`` parameter.
124
