233 class Configurable(object):
234 """Base class for configurable interfaces.
235  
236 A configurable interface is an (abstract) class whose constructor
237 acts as a factory function for one of its implementation subclasses.
238 The implementation subclass as well as optional keyword arguments to
239 its initializer can be set globally at runtime with `configure`.
240  
241 By using the constructor as the factory method, the interface
242 looks like a normal class, `isinstance` works as usual, etc. This
243 pattern is most useful when the choice of implementation is likely
244 to be a global decision (e.g. when `~select.epoll` is available,
245 always use it instead of `~select.select`), or when a
246 previously-monolithic class has been split into specialized
247 subclasses.
248  
249 Configurable subclasses must define the class methods
250 `configurable_base` and `configurable_default`, and use the instance
251 method `initialize` instead of ``__init__``.
252  
253 .. versionchanged:: 5.0
254  
255 It is now possible for configuration to be specified at
256 multiple levels of a class hierarchy.
257
