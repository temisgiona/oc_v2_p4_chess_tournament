32 class MonitoredQueue(MonitoredQueueBase, Proxy):
33 """Class for running monitored_queue in the background.
34  
35 See zmq.devices.Device for most of the spec. MonitoredQueue differs from Proxy,
36 only in that it adds a ``prefix`` to messages sent on the monitor socket,
37 with a different prefix for each direction.
38  
39 MQ also supports ROUTER on both sides, which zmq.proxy does not.
40  
41 If a message arrives on `in_sock`, it will be prefixed with `in_prefix` on the monitor socket.
42 If it arrives on out_sock, it will be prefixed with `out_prefix`.
43  
44 A PUB socket is the most logical choice for the mon_socket, but it is not required.
