12 class _allow_interrupt(object):
13 """Utility for fixing CTRL-C events on Windows.
14  
15 On Windows, the Python interpreter intercepts CTRL-C events in order to
16 translate them into ``KeyboardInterrupt`` exceptions. It (presumably)
17 does this by setting a flag in its "console control handler" and
18 checking it later at a convenient location in the interpreter.
19  
20 However, when the Python interpreter is blocked waiting for the ZMQ
21 poll operation to complete, it must wait for ZMQ's ``select()``
22 operation to complete before translating the CTRL-C event into the
23 ``KeyboardInterrupt`` exception.
24  
25 The only way to fix this seems to be to add our own "console control
26 handler" and perform some application-defined operation that will
27 unblock the ZMQ polling operation in order to force ZMQ to pass control
28 back to the Python interpreter.
29  
30 This context manager performs all that Windows-y stuff, providing you
31 with a hook that is called when a CTRL-C event is intercepted. This
32 hook allows you to unblock your ZMQ poll operation immediately, which
33 will then result in the expected ``KeyboardInterrupt`` exception.
34  
35 Without this context manager, your ZMQ-based application will not
36 respond normally to CTRL-C events on Windows. If a CTRL-C event occurs
37 while blocked on ZMQ socket polling, the translation to a
38 ``KeyboardInterrupt`` exception will be delayed until the I/O completes
39 and control returns to the Python interpreter (this may never happen if
40 you use an infinite timeout).
41  
42 A no-op implementation is provided on non-Win32 systems to avoid the
43 application from having to conditionally use it.
44  
45 Example usage:
46  
47 .. sourcecode:: python
48  
49 def stop_my_application():
50 # ...
51  
52 with allow_interrupt(stop_my_application):
53 # main polling loop.
54  
55 In a typical ZMQ application, you would use the "self pipe trick" to
56 send message to a ``PAIR`` socket in order to interrupt your blocking
57 socket polling operation.
58  
59 In a Tornado event loop, you can use the ``IOLoop.stop`` method to
60 unblock your I/O loop.
61 """
