57 def forbid_nonstring_types(forbidden, name=None):
58 """
59 Decorator to forbid specific types for a method of StringMethods.
60  
61 For calling `.str.{method}` on a Series or Index, it is necessary to first
62 initialize the :class:`StringMethods` object, and then call the method.
63 However, different methods allow different input types, and so this can not
64 be checked during :meth:`StringMethods.__init__`, but must be done on a
65 per-method basis. This decorator exists to facilitate this process, and
66 make it explicit which (inferred) types are disallowed by the method.
67  
68 :meth:`StringMethods.__init__` allows the *union* of types its different
69 methods allow (after skipping NaNs; see :meth:`StringMethods._validate`),
70 namely: ['string', 'empty', 'bytes', 'mixed', 'mixed-integer'].
71  
72 The default string types ['string', 'empty'] are allowed for all methods.
73 For the additional types ['bytes', 'mixed', 'mixed-integer'], each method
74 then needs to forbid the types it is not intended for.
75  
76 Parameters
77 ----------
78 forbidden : list-of-str or None
79 List of forbidden non-string types, may be one or more of
80 `['bytes', 'mixed', 'mixed-integer']`.
81 name : str, default None
82 Name of the method to use in the error message. By default, this is
83 None, in which case the name from the method being wrapped will be
84 copied. However, for working with further wrappers (like _pat_wrapper
85 and _noarg_wrapper), it is necessary to specify the name.
86  
87 Returns
88 -------
89 func : wrapper
90 The method to which the decorator is applied, with an added check that
91 enforces the inferred type to not be in the list of forbidden types.
92  
93 Raises
94 ------
95 TypeError
96 If the inferred type of the underlying data is in `forbidden`.
97 """
